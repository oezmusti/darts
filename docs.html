1. Überblick

Projektziele

Die Darts Tracker App ermöglicht es, Dart-Spiele digital zu verfolgen, zu speichern und die Statistiken zu analysieren.

Hauptfunktionalitäten

  • ✅ Spielverwaltung: Neue Spiele starten mit bis zu 4 Spielern
  • ✅ Verschiedene Spielmodi: 501er, 301er, 101er oder benutzerdefinierte Startpunkte
  • ✅ Live-Tracking: Echtzeit-Verfolgung der Punkte pro Spieler
  • ✅ Wurf-Eingabe: Intuitives Dartbrett mit Multiplier-Optionen
  • ✅ Bust-Erkennung: Automatische Erkennung ungültiger Züge
  • ✅ Checkout-Berechnung: Intelligente Vorschläge für Finishing-Würfe
  • ✅ Spiel-Historie: Persistente Speicherung aller gespielten Spiele
  • ✅ Responsive Design: Funktioniert auf Desktop, Tablet und Mobilgeräten

Technologie-Stack

Kategorie Technologie
Frontend HTML5, CSS3, Vanilla JavaScript
Persistierung Browser LocalStorage API
Architektur Modulares JavaScript Pattern (IIFE)
Datenformat JSON

🎮 game.js - Spiel-Logik

Zweck: Verwaltung des Spielstatus, Punkt-Berechnung und Checkout-Strategien

Funktion Beschreibung Parameter
initGame() Initialisiert ein neues Spiel title, startingPoints, playerNames
getCurrentPlayer() Gibt den aktuellen Spieler zurück keine
submitThrow() Verarbeitet einen Wurf eines Spielers totalPoints, throwsDetails[]
nextPlayer() Wechselt zum nächsten aktiven Spieler keine
calculateCheckouts() Berechnet mögliche Finishing-Würfe remainingPoints
endGame() Beendet das aktuelle Spiel keine

Kern-Game-Logik:

gameState = { title: String, // Spiel-Name startingPoints: Number, // Start-Punkte (z.B. 501) currentRound: Number, // Aktuelle Runde players: [ // Array von Spielern { name: String, currentPoints: Number, throws: [], // Array aller Würfe finished: Boolean, finishedRank: Number } ], currentPlayerIndex: Number, // Index des aktiven Spielers finishedOrder: [], // Reihenfolge der Beendigung history: [] // Spiel-Historie };
Game.submitThrow(totalPoints, throwsDetails)
Hauptfunktion zur Verarbeitung von Würfen
Logik:
1. Prüfe auf Bust (Punkte < 0)
2. Ziehe Punkte ab
3. Speichere Wurf-Details
4. Prüfe auf Spielende (genau 0 Punkte)
5. Rückgabe: { success, message, gameOver? }
Game.calculateCheckouts(remainingPoints)
Intelligente Checkout-Berechnung für letzte Würfe
Algorithmus:
- Alle möglichen 3er-Dart-Kombinationen durchprobieren
- Der letzte Dart MUSS ein Double sein
- Bullseye (50 Punkte) ist möglich
- Max. 20 Optionen anzeigen

4. Workflows & Prozesse

Workflow 1: Neues Spiel starten

1️⃣ Nutzer klickt "Neues Spiel Starten"
↓
2️⃣ navigateTo('setup') wird aufgerufen
↓
3️⃣ Setup-Formular wird angezeigt
↓
4️⃣ Nutzer füllt aus: Titel, Startpunkte, Spieler
↓
5️⃣ Game.initGame() wird aufgerufen
↓
6️⃣ Spiel-Seite wird angezeigt

Workflow 2: Wurf-Verarbeitung

1️⃣ Spieler wählt Dartbrett-Nummer
↓
2️⃣ UI.selectThrow() speichert Wurf
↓
3️⃣ Spieler wählt Multiplier (Single/Double/Triple)
↓
4️⃣ UI.updateThrowsDisplay() zeigt Wurf-Auswahl
↓
5️⃣ Spieler klickt Bestätigung
↓
6️⃣ Game.submitThrow() verarbeitet
↓
7️⃣ Bust-Prüfung erfolgt
↓
8️⃣ Punkte werden aktualisiert
↓
9️⃣ Game.nextPlayer() wechselt Spieler

Workflow 3: Spiel beenden & Speichern

1️⃣ Ein Spieler erreicht genau 0 Punkte
↓
2️⃣ Spieler wird in finishedOrder registriert
↓
3️⃣ Wenn alle Spieler fertig: Spiel beendet
↓
4️⃣ Game.endGame() wird aufgerufen
↓
5️⃣ Storage.saveGame() speichert alles
↓
6️⃣ Ergebnis-Screen wird angezeigt

Wurf-Verarbeitungs-Details:

Wurf-Details Struktur: { baseValue: Number, // z.B. 20 multiplier: 1|2|3, // Single (1), Double (2), Triple (3) totalPoints: Number // baseValue * multiplier } Beispiele: - Single 20 = 20 Punkte - Double 20 = 40 Punkte (D20) - Triple 20 = 60 Punkte (T20) - Bullseye außen = 25 Punkte (nur Single) - Bullseye innen = 50 Punkte (nur Single)

7. Technische Details

Modul-Pattern (IIFE)

Alle Module verwenden das Immediately Invoked Function Expression (IIFE) Pattern:

const ModuleName = (() => { // Private Variables let privateVar = {}; // Private Functions function privateFunction() { } // Public Interface (Return) return { publicMethod() { }, publicProperty: value }; })(); Vorteile: ✓ Kapslung (private & public Members) ✓ Namespace-Schutz (keine globalen Variablen) ✓ Sichere Kommunikation zwischen Modulen ✓ Leichte Wartung und Tests

Event-Handling

Event Trigger Handler
click (Dartbrett) Spieler klickt Zahl UI.selectThrow()
click (Multiplier) Spieler klickt Single/Double/Triple onMultiplierChange()
click (Bestätigung) Spieler klickt ✓ Game.submitThrow()
click (Löschen) Spieler klickt Wurf selectedThrows.splice()
change (Setup) Startpunkte ändern handleCustomPointsChange()

Error Handling

Fehlerbehandlung:
❌ Bust: submitThrow() gibt { success: false } zurück
❌ LocalStorage voll: Try-Catch in Storage Module
❌ Ungültige Eingaben: Form Validation HTML5

Performance-Optimierungen

  • ✅ Checkout-Berechnung limitiert auf 20 Vorschläge
  • ✅ DOM-Manipulation nur bei Bedarf
  • ✅ Event-Listeners mit stopPropagation()
  • ✅ LocalStorage JSON Stringification nur beim Speichern

Dart-Board Implementierung

DART_BOARD = [ 20, 1, 18, 4, 13, 6, 10, 15, 2, 17, 3, 19, 7, 16, 8, 11, 14, 9, 12, 5 ] Diese Anordnung ist die Standard Dart-Board Reihenfolge im echten Spiel. Zusätzliche Werte: - 25 = Außen-Bullseye (nur Single möglich) - 50 = Innen-Bullseye / Bull (nur Single möglich) - 0 = Miss (kein Treffer)

Multiplier-System

Multiplier Name Formel Beispiel
1 Single Zahl × 1 20 = 20 Punkte
2 Double (D) Zahl × 2 D20 = 40 Punkte
3 Triple (T) Zahl × 3 T20 = 60 Punkte

Checkout-Berechnung Algorithmus

Eingabe: remainingPoints (z.B. 79) Algorithmus: 1. Iteriere durch alle 20 Dartbrett-Zahlen 2. Für jeden 1. Dart: multiplier 1-3 (Single/Double/Triple) 3. Für jeden 2. Dart: multiplier 1-3 4. Für jeden 3. Dart: multiplier MUSS 2 sein (Double) Bedingungen: - first + second < remainingPoints - first + second + third = remainingPoints - Bullseye kann auch passieren Ergebnis: Array möglicher Dart-Kombinationen (max. 20 Stück)

Player-Wechsel Logik

// Wechsel zum nächsten Spieler nextPlayer() { let nextIndex = (currentIndex + 1) % players.length; let attempts = 0; // Überspringe fertige Spieler while (players[nextIndex].finished && attempts < players.length) { nextIndex = (nextIndex + 1) % players.length; attempts++; } currentPlayerIndex = nextIndex; } Nutzen: % Operator für Zirkular-Liste Sicherheit: Schleife bricht nach max. attempts ab

Typischerweise 5-10 MB pro Domain
Für ca. 100-200 Spiele auf einem Gerät ausreichend
